# LightSwitch — Технічне завдання

## 1. Загальний опис

**LightSwitch** — легка, швидка та надійна програма для Windows, яка дозволяє:
- Перемикати розкладку клавіатури виділеною клавішею/комбінацією для кожної мови
- Конвертувати вже набраний текст між розкладками

## 2. Технологічний стек

| Компонент | Технологія | Обґрунтування |
|-----------|-----------|---------------|
| Мова | **Rust** | Компілюється в нативний код, мінімальний runtime, безпека пам'яті, один бінарник без залежностей |
| Win32 API | Крейт `windows` (офіційний від Microsoft) | Прямий доступ до WinAPI без обгорток |
| UI (налаштування) | `native-windows-gui` або raw Win32 через `windows` | Найшвидший відгук, нативний вигляд, мінімальний розмір |
| Tray icon | Win32 `Shell_NotifyIconW` | Нативно, без залежностей |
| Збірка | `cargo build --release` | Один виконуваний файл, standalone |

**Цільова платформа (MVP):** Windows 10+ x86-64

## 4. Функціональні вимоги

### 4.1. Перехоплення клавіш

- **Механізм:** `SetWindowsHookExW` з `WH_KEYBOARD_LL` (не потребує адмінських прав)
- **Обробка модифікаторів:** Hook відстежує стан Ctrl, Shift, Alt. Якщо натиснуто модифікатор + інша клавіша — пропускаємо (не перехоплюємо). Якщо натиснуто лише призначену клавішу/комбінацію — виконуємо дію
- **Латентність:** Обробка в hook callback має бути мінімальною. Важкі операції (конвертація, clipboard) виконуються через `PostMessage` у головний потік

### 4.2. Перемикання розкладки

- Для кожної встановленої розкладки — окремий hotkey, що призначається користувачем
- Перемикання через `PostMessage(hwnd, WM_INPUTLANGCHANGEREQUEST, ...)`
- Або через Windows API `ActivateKeyboardLayout`

### 4.3. Конвертація тексту

**Алгоритм:**
1. Зберегти поточний вміст clipboard
2. Якщо є виділений текст — скопіювати його (`Ctrl+C` симуляція)
3. Якщо виділення порожнє — виділити весь текст (`Ctrl+A`), потім скопіювати
4. Прочитати текст з clipboard
5. Конвертувати послідовно через ланцюг розкладок (поточна → наступна), поки результат не стане валідним або не пройде повне коло
6. Вставити результат (`Ctrl+V` симуляція)
7. Якщо було виконано `Ctrl+A` — зняти виділення (натиснути `→` або `End`)
8. Відновити попередній вміст clipboard

**Таблиці відповідності:**
- Зберігаються як `HashMap<(u16, char), char>` — `(layout_id, символ) → символ_в_іншій_розкладці`
- Базові таблиці EN↔RU, EN↔UA, RU↔UA вбудовані
- Можливість додавання користувацьких таблиць у майбутньому

**Циклічна конвертація:**
- Приклад для 3 розкладок (EN, RU, UA): `Ghbdsn` → `Привыт` → `Привіт`
- Кожне натискання hotkey конвертації переводить текст у наступну розкладку в ланцюгу
- Порядок визначається порядком розкладок у системі (або налаштуваннях)

### 4.4. Визначення встановлених розкладок

- При запуску — виклик `GetKeyboardLayoutList` для отримання всіх HKL
- Маппінг HKL → назва мови через `GetLocaleInfoW`
- Автоматичне оновлення при зміні списку розкладок у системі (WM_INPUTLANGCHANGE або періодичне опитування)
- Fallback: ручне додавання через UI (кнопка "+", випадаючий список)

### 4.5. Вікно налаштувань

**Структура:**

```
╔══════════════════════════════════════════╗
║  LightSwitch — Налаштування             ║
╠══════════════════════════════════════════╣
║                                          ║
║  Розкладки:                              ║
║  ┌──────────┬────────────────────────┐   ║
║  │ English  │ [натисніть клавішу...] │   ║
║  │ Русский  │ [Ctrl+1             ]  │   ║
║  │ Українська│ [Ctrl+2             ] │   ║
║  └──────────┴────────────────────────┘   ║
║  ⚠ Ctrl+1 вже використовується!          ║
║                                          ║
║  Конвертація тексту:                     ║
║  Гаряча клавіша: [Pause/Break        ]   ║
║                                          ║
║  ☐ Конвертувати лише виділений текст      ║
║    (якщо вимкнено — Ctrl+A + конвертація) ║
║                                          ║
║  ─────────────────────────────────────   ║
║  ☑ Запускати при старті Windows           ║
║    ○ Тільки для мене                      ║
║    ○ Для всіх користувачів (потрібні      ║
║      права адміністратора)                ║
║                                          ║
║         [ Зберегти ]  [ Скасувати ]      ║
╚══════════════════════════════════════════╝
```

**Призначення клавіш:**
- Користувач клікає поле → поле переходить у режим "очікування" → натискає бажану клавішу/комбінацію → вона відображається в полі
- Перевірка конфліктів через `RegisterHotKey` (спроба зареєструвати → якщо помилка → клавіша зайнята → візуальне попередження: червона рамка + текст під полем)
- Після успішної перевірки — `UnregisterHotKey` тестової реєстрації

### 4.6. Tray icon

- Іконка в системному треї з контекстним меню:
  - **Налаштування** — відкриває вікно конфігурації
  - **Поточна розкладка** — інформаційний рядок
  - **Вихід** — завершення програми
- Подвійний клік — відкриття налаштувань

### 4.7. Автозапуск

- **Для поточного користувача:** запис у `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` (не потребує адмінських прав, без попереджень безпеки)
- **Для всіх користувачів:** запис у `HKLM\Software\Microsoft\Windows\CurrentVersion\Run` (потребує UAC elevation)
- Галочка в налаштуваннях + радіокнопки вибору scope

### 4.8. Конфігурація

- Формат: **TOML** (стандарт у Rust-екосистемі, людиночитабельний)
- Розташування: `%APPDATA%\LightSwitch\config.toml`
- Структура:

```toml
[general]
autostart = true
autostart_scope = "user"  # "user" | "all_users"

[layouts]
# layout_id = "hotkey"
"00000409" = "CapsLock"        # English
"00000419" = "Shift+CapsLock"  # Russian
"00000422" = "Alt+CapsLock"    # Ukrainian

[conversion]
hotkey = "Pause"
mode = "auto"  # "auto" (виділене або все) | "selection_only"
```

## 5. Нефункціональні вимоги

| Вимога | Критерій |
|--------|---------|
| Розмір бінарника | < 5 MB (ідеально < 2 MB) |
| Споживання RAM | < 10 MB у стані спокою |
| Латентність hook | < 1 мс на обробку натискання |
| Латентність конвертації | < 100 мс для тексту до 10K символів |
| Standalone | Один .exe файл, без зовнішніх DLL |
| Без адмін прав | Все окрім "автозапуск для всіх" працює без elevation |

## 7. Підтримувані клавіші для hotkey

### Standalone модифікатори (натиснув → відпустив без інших клавіш)

| Клавіша | Підтримка | Примітка |
|---------|:---------:|---------|
| LCtrl | ✅ | Розрізняється лівий/правий |
| RCtrl | ✅ | Розрізняється лівий/правий |
| LShift | ✅ | Розрізняється лівий/правий |
| RShift | ✅ | Розрізняється лівий/правий |
| LAlt | ✅ | Розрізняється лівий/правий |
| CapsLock | ✅ | Як звичайна клавіша (не модифікатор) |

### Клавіші, що НЕ підтримуються / відкладені

| Клавіша | Причина |
|---------|---------|
| RAlt | На багатьох розкладках RAlt = AltGr. Windows надсилає Ctrl+Alt замість RAlt. Потребує окремої обробки |
| LWin / RWin | Hook бачить натискання, але Windows перехоплює для Start Menu. Suppress може зламати Win+комбінації |
| Fn | Обробляється firmware клавіатури, Windows API не бачить цю клавішу |

### Комбінації з модифікаторами

Будь-яка звичайна клавіша з Ctrl/Shift/Alt: Ctrl+1, Shift+F1, Alt+Space тощо.
Standalone модифікатор коректно скасовується, якщо під час утримання натиснуто будь-яку іншу клавішу (Ctrl+C, Ctrl+Shift тощо працюють штатно).

## 8. Подібні програми

1. **Punto Switcher** (puntos.ru) — має майже весь потрібний функціонал, але фактично є клавіатурним шпигуном
2. **Mahou** (github.com/iamkarlson/Mahou) — має майже весь функціонал, але надмірно складна
3. **dotSwitcher**, **AltSwitcher** — виглядають закинутими
4. **SimpleSwitcher** (github.com/Aegel5/SimpleSwitcher) — в активній розробці, але неповний функціонал
5. **Faine Switcher** (faineswitcher.com) — неповний функціонал, ймовірно закинута

Загальна тенденція: або велетенський комбайн з прихованим шпіонським модулем, або щось сире чи закинуте.

## 9. Відомі ризики та обмеження

1. **Clipboard race condition** — деякі програми можуть конкурувати за clipboard. Мітигація: невелика затримка між операціями (10-50 мс)
2. **UAC-захищені вікна** — hook не працює для процесів з вищим рівнем прав. У прототипі не підтримується. Пізніше — опція в налаштуваннях для запуску з elevation
3. **Таблиці конвертації** — маппінг символів між розкладками. Таблиці компактні (~100-150 пар на розкладку, кілька KB), підхід "в лоб" цілком робочий і надійний. Включають символи з Shift (великі літери, спецсимволи)
4. **Антивіруси** — keyboard hooks можуть викликати спрацювання. Можливо знадобиться підписання бінарника
